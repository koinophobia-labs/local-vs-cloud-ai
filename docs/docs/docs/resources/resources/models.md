# Models

AI architecture depends heavily on which models are available and where they can run.

Some models are designed mainly for cloud use.

Others can run locally or on private infrastructure depending on size, hardware, and setup.

## Local and Self-Hosted Models

These model families are commonly used for local or self-hosted AI experimentation:

* Llama
* Mistral
* Gemma
* Qwen
* Phi

These models can support:

* Local assistants
* Private document analysis
* Offline experimentation
* Self-hosted workflows
* Cost-controlled AI systems

## Cloud Models

These model families are commonly accessed through hosted platforms and APIs:

* GPT
* Claude
* Gemini
* Grok

These models can support:

* Advanced reasoning
* Coding assistance
* Content generation
* Research workflows
* Customer-facing AI products
* Large-scale applications

## Choosing a Model

Model choice depends on:

* Task complexity
* Privacy requirements
* Hardware limits
* Cost
* Latency
* Reliability
* Developer experience

## Local Model Fit

Local models are often useful when:

* Data should stay private
* Repeated usage would create high API costs
* Offline access matters
* The task is simple enough for smaller models
* The builder wants more control

## Cloud Model Fit

Cloud models are often useful when:

* The task requires stronger reasoning
* The product needs fast deployment
* The system must scale across many users
* Hardware should not be managed by the builder
* The best available model quality matters most

## Core Takeaway

The strongest AI systems do not always use the biggest model.

They use the right model in the right place for the right job.
