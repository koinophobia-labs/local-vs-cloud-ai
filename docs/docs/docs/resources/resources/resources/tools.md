# Tools

Models provide intelligence.

Tools provide the infrastructure that allows AI systems to run, communicate, and interact with users.

## Local AI Tools

### Ollama

A popular framework for running AI models locally.

Common uses:

* Local chat assistants
* Private workflows
* Self-hosted AI systems

### LM Studio

A desktop application for running and testing local AI models.

Common uses:

* Model experimentation
* Prompt testing
* Local development

### Open WebUI

A self-hosted interface for local and hybrid AI systems.

Common uses:

* Team environments
* Internal assistants
* Private deployments

## Cloud AI Platforms

### OpenAI API

Provides access to GPT models through hosted infrastructure.

### Anthropic API

Provides access to Claude models through hosted infrastructure.

### Gemini API

Provides access to Google's Gemini models.

## Hybrid AI Infrastructure

### LangGraph

Framework for building agent workflows and stateful AI systems.

### Vercel AI SDK

Tools for integrating AI into modern web applications.

### OpenRouter

Provides access to multiple model providers through a unified API.

## Core Takeaway

A successful AI system depends on more than choosing a model.

The surrounding tools often determine deployment speed, reliability, scalability, and developer experience.
